# Support

## Community Support

Use public issues for reproducible bugs, documentation gaps, and feature
requests. Include enough detail for maintainers and contributors to reproduce the
problem.

Good reports include:

- PolyTalk commit or release
- Deployment mode: local Python, Docker CPU, Docker GPU, or external services
- STT, translation, and TTS provider/model names
- Relevant `.env` and `config/config.yaml` values with secrets removed
- Browser and operating system
- Logs around the failure
